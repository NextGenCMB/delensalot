data and simulations access

This section describes how to use simulation modules used for lensing reconstruction. At the end, you will understand the structure of the data file, adapt it to your liking, and create your very own to read in your data.

A complete list of DLENSALOT_Data parameters are shown in the configuration files section.

delensalot uses a single T,E, and B map to perform lensing reconstruction. They can come both, in spherical harmonics, or pixel space. We assume the data to be in HealPix-format.

Each class of a simulation module defines a simulation set. For each simulation set, class parameters configure the simulation set and delensalot provides ways to handle this.

Each class must contain the following two functions to work for lensing reconstruction:

  • hashdict(), return a dictionary that describes the (intermediate) data stored on disk,

  • <get_sim_map>(), returns the simulation map

All other functions are optional; vanilla delensalot will not use them.

minmal working example

The minimal working example generate simulation generates T,E, and B spherical harmonics from a FFP10 power spectrum using the delensalot-internal simulation library, is lensed using lenspyx, and Gaussian noise is added. The full simulated polarization maps inclusive of the transfer function comes from plancklens.

The noise level can be set via the parameter nlev.

 1"""
 2Simulates CMB polarization maps generated on the fly, inclusive of isotropic white noise.
 3"""
 4
 5import numpy as np
 6import os
 7import delensalot
 8from delensalot import utils
 9from os.path import join as opj
10
11from delensalot.config.metamodel.dlensalot_mm import *
12
13
14dlensalot_model = DLENSALOT_Model(
15    defaults_to = 'P_FS_CMBS4',
16
17    job = DLENSALOT_Job(
18        jobs = ["generate_sim"]
19    ),                             
20    analysis = DLENSALOT_Analysis(
21        key = 'p_p',
22        simidxs = np.arange(0,1),
23        TEMP_suffix = 'my_fist_delensalot_analysis_simgen',
24    ),
25    data = DLENSALOT_Data(
26        package_ = 'delensalot',
27        module_ = 'sims.generic',
28        class_ = 'sims_cmb_len',
29        class_parameters = {
30            'lmax': 5976,
31            'cls_unl': utils.camb_clfile(opj(os.path.dirname(delensalot.__file__, 'data', 'cls'), 'FFP10_wdipole_lenspotentialCls.dat')),
32            'lib_dir': opj(os.environ['SCRATCH'], 'sims', 'generic', 'nside2048', 'lmax5976', 'nlevp_sqrt(2)')
33        },
34        nlev_t = 1.00,
35        nlev_p = np.sqrt(2),
36        beam = 1.00,
37        lmax_transf = 3000,
38        nside = 2048,
39    )
40)

build your own

To use your own data or simulations, you will have to write a class that provides the function get_sim_pmap() and point delensalot to it. Your own simulation module can, in principle, be put anywhere.
